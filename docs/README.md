## 📝 Contributing to Docs

When adding new documentation:
- **Guides** → `docs/guides/` - How-to guides and tutorials
- **Architecture** → `docs/architecture/` - System design and technical decisions
- **Plans** → `docs/plans/` - Feature planning and roadmaps
- **Design** → `docs/design/` - UI/UX and design system docs
- **Archive** → `docs/archive/` - Historical/completed documentation

Keep documentation:
- **Clear and concise**
- **Up to date** - remove or archive outdated docs
- **Well-organized** - use this structure
- **Cross-referenced** - link related documents
